Question 1 code from memory

Write a FastAPI GET endpoint that takes two integers as input and returns a dictionary containing their sum and their product.

Assume the usual imports and that the FastAPI app has already been created as app. Write just the endpoint function (including its decorator).

Answer
@app.get("/compute")
def compute(a: int, b: int):
    return {"sum": a + b, "product": a * b}

Reasoning:

  • @app.get("/compute") registers the function as a handler for GET requests at the path /compute.
  • Because a and b are typed as int and are not part of the path string, FastAPI treats them as query parameters.
  • Returning a Python dict causes FastAPI to serialize the response as JSON, e.g. {"sum": 7, "product": 12}.
Question 2 short answer

Suppose the FastAPI app from the previous question is running, and call its base URL homepage. Write the full URL a client would use to call the /compute endpoint with $a = 3$ and $b = 4$, and state the JSON response the server would return.

Answer

URL:

homepage/compute?a=3&b=4

JSON response:

{"sum": 7, "product": 12}

Reasoning:

  • Query parameters are appended to the path after a ?, with each parameter written as name=value and multiple parameters separated by &.
  • So a = 3 and b = 4 become ?a=3&b=4.
  • The endpoint returns {"sum": a + b, "product": a * b}, which evaluates to {"sum": 7, "product": 12}.
Question 3 code from memory

Write a FastAPI GET endpoint that takes a string as input and returns the encoding of that string as a list (of token IDs). You may assume that a dictionary named vocab_dictionary, mapping each token in the vocabulary to its integer index, has already been defined.

Assume the usual imports and that the FastAPI app has already been created as app. Write just the endpoint function (including its decorator).

Answer
@app.get("/encode")
def encode(s: str):
    encoded = []
    for c in s:
        encoded.append(vocab_dictionary[c])
    return encoded

Reasoning:

  • @app.get("/encode") registers the function as a handler for GET requests at the path /encode.
  • Because s is typed as str and is not part of the path, FastAPI treats it as a query parameter.
  • Inside the function, we loop through each token c in the input string, look up its integer index in vocab_dictionary, and append it to the list encoded.
  • Returning a Python list causes FastAPI to serialize the response as a JSON array.
Question 4 code from memory

Suppose the following have already been defined in the FastAPI app: 

vocab = ["the", "cat", "dog", "sat", "ran", "on", "mat"]
embedding = nn.Embedding(7, 2)

Write a FastAPI GET endpoint that returns a dictionary mapping each vocabulary index to its embedded vector (as a list of two numbers).

Assume the usual imports (including torch) and that the FastAPI app has been created as app. Write just the endpoint function (including its decorator).

Answer
@app.get("/embeddings")
def get_embeddings():
    result = {}
    for i in range(len(vocab)):
        result[i] = embedding.weight[i].tolist()
    return result

Reasoning:

  • @app.get("/embeddings") registers the function as the handler for GET /embeddings. The endpoint takes no inputs.
  • The size of the vocabulary is len(vocab), so we loop over indices i = 0, 1, ..., len(vocab) - 1.
  • embedding.weight is a tensor of shape $(7, 2)$ holding the embedding table, so embedding.weight[i] is row $i$ — the 2-dimensional embedded vector for token $i$.
  • .tolist() converts that tensor into a plain Python list so FastAPI can serialize it as JSON (PyTorch tensors are not JSON-serializable on their own).
  • Returning the dictionary produces a JSON object of the form {"0": [x0, y0], "1": [x1, y1], ..., "6": [x6, y6]}.
Question 5 code from memory

Write a FastAPI GET endpoint that takes a string of text as input and returns the embedding of that text as a list of lists (one 2-dimensional vector per token).

You may assume that vocab_dictionary (mapping each token to its index) and embedding (an nn.Embedding layer) have already been defined, and that the usual imports (including torch) are available.

Answer
@app.get("/embed_text")
def embed_text(s: str):
    result = []
    for c in s:
        i = vocab_dictionary[c]
        result.append(embedding.weight[i].tolist())
    return result

Reasoning:

  • @app.get("/embed_text") registers the function as the handler for GET /embed_text.
  • Because s is typed as str and is not part of the path, FastAPI treats it as a query parameter.
  • For each token c in the input string, we look up its index i in vocab_dictionary, grab the corresponding row of the embedding table with embedding.weight[i], and convert that 2-D tensor to a list with .tolist().
  • Appending each such 2-element list to result builds a list of lists whose shape is (length of text) $\times$ 2.
  • Returning a Python list of lists causes FastAPI to serialize the response as a JSON array of arrays.
