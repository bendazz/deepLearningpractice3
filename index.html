Question 1 fill in the blanks

You are building a feed-forward network in PyTorch to classify color (RGB) images of size $40 \times 56$ pixels into 7 classes. Fill in every blank ( ) so that the architecture is valid and appropriate for this task. 

model = nn.Sequential(
    nn.Flatten(),
    nn.Linear( , 256),
    nn.ReLU(),
    nn.Linear( , 64),
    nn.ReLU(),
    nn.Linear( ,  )
)
Answer
model = nn.Sequential(
    nn.Flatten(),
    nn.Linear(6720, 256),
    nn.ReLU(),
    nn.Linear(256, 64),
    nn.ReLU(),
    nn.Linear(64, 7)
)

Reasoning:

  • Each image has shape (channels, height, width) $= (3, 40, 56)$. After Flatten, it becomes a vector of length $3 \times 40 \times 56 = 6720$, so the first Linear takes 6720 inputs.
  • The input dimension of each subsequent Linear must match the output dimension of the previous one: $256 \to 256$, then $64 \to 64$.
  • The final layer must output one score per class, so the output dimension is 7.
Question 2 fill in the blanks

You are now building a network with two convolutional layers (each followed by a $2 \times 2$ max-pool) in the front, followed by a small feed-forward classifier. The input is color (RGB) images of size $24 \times 36$ pixels, and there are 5 classes. Every Conv2d uses a $3 \times 3$ kernel with padding=1, so the convolutions do not change the spatial dimensions — only the pooling layers do. Fill in every blank ( ). 

model = nn.Sequential(
    nn.Conv2d( , 16, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.MaxPool2d(2),
    nn.Conv2d( , 32, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.MaxPool2d(2),
    nn.Flatten(),
    nn.Linear( , 128),
    nn.ReLU(),
    nn.Linear( ,  )
)
Answer
model = nn.Sequential(
    nn.Conv2d(3, 16, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.MaxPool2d(2),
    nn.Conv2d(16, 32, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.MaxPool2d(2),
    nn.Flatten(),
    nn.Linear(1728, 128),
    nn.ReLU(),
    nn.Linear(128, 5)
)

Reasoning: track the tensor shape at each step, written as (channels, height, width).

  • Input image: $(3, 24, 36)$. So the first Conv2d's in_channels is 3.
  • After the first Conv2d (padding preserves spatial dims): $(16, 24, 36)$.
  • After the first MaxPool2d(2): $(16, 12, 18)$.
  • The second Conv2d's in_channels must match the current channel count, which is 16.
  • After the second Conv2d: $(32, 12, 18)$.
  • After the second MaxPool2d(2): $(32, 6, 9)$.
  • After Flatten, each sample is a vector of length $32 \times 6 \times 9 = 1728$, which is the input size of the first Linear.
  • The next Linear's input must match the previous one's output (128), and its output must equal the number of classes, 5.

Write the prediction vector the network produces for this batch.

Answer

Prediction vector: $[\,3,\ 7,\ 0,\ 9\,]$

  • Row 0: the maximum is $4.1$ at column 3 $\Rightarrow$ prediction 3.
  • Row 1: the maximum is $5.6$ at column 7 $\Rightarrow$ prediction 7.
  • Row 2: the maximum is $3.7$ at column 0 $\Rightarrow$ prediction 0.
  • Row 3: the maximum is $3.6$ at column 9 $\Rightarrow$ prediction 9.

The true labels for these 6 samples are:

Answer

Total accuracy = 4 correct out of 6.

Reasoning: take the $\operatorname{argmax}$ of each row to get the prediction vector, then compare to the targets:

  • Row 0: max $3.9$ at column 4 $\Rightarrow$ pred 4, target 4  ✓

  • Row 2: max $5.3$ at column 2 $\Rightarrow$ pred 2, target 8  ✗

  • Row 4: max $4.0$ at column 1 $\Rightarrow$ pred 1, target 1  ✓
  • Row 5: max $3.4$ at column 7 $\Rightarrow$ pred 7, target 3  ✗

So predictions $= [4, 6, 2, 5, 1, 7]$ and 4 of them match the target vector.

Question 5 code from memory

Assume the following have already been defined:

  • model — a PyTorch neural network
  • loader — a DataLoader that yields (images, labels) batches
  • criterion — a loss function (e.g., nn.CrossEntropyLoss())
  • optimizer — an optimizer attached to model.parameters()
  • epochs — an integer number of training epochs

Write a PyTorch training loop that trains model for epochs epochs. You do not need to track loss or accuracy, evaluate on a test set, or print anything — just the training itself.

Answer
for epoch in range(epochs):
    for images, labels in loader:
        optimizer.zero_grad()
        output = model(images)
        loss = criterion(output, labels)
        loss.backward()
        optimizer.step()

What each line does:

  • optimizer.zero_grad(): clears gradients from the previous batch (PyTorch accumulates them by default).
  • output = model(images): forward pass — compute predictions for this batch.
  • loss = criterion(output, labels): compare predictions to true labels to get a scalar loss.
  • loss.backward(): backpropagate — compute the gradient of the loss with respect to every parameter.
  • optimizer.step(): update the parameters using those gradients.
